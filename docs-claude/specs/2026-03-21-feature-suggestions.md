# MEMON 기능 개선 및 신규 기능 제안서

**작성일:** 2026-03-21

---

## 현재 상태 요약

MEMON webadmin은 경조사 기록 관리 앱으로, 현재 다음 기능이 구현되어 있습니다:
- **대시보드**: 환영 메시지, 요약 통계(보낸/받은 금액, 차액), 다가오는 일정 테이블, 최근 기록 테이블
- **관계 관리**: 카드형 목록, CRUD, 관계 유형/태그 관리, 상세 페이지(기록/일정 탭)
- **기록 관리**: 테이블형 목록, 검색/필터/정렬/페이지네이션, 상세 페이지
- **경조사 관리**: 카드형 목록, 유형 필터 탭, 상세 페이지
- **일정 관리**: 다가오는 일정 뷰 + 캘린더(리스트형) 뷰 전환
- **통계**: 월별 추이(바 형태), 기록 유형/관계 유형 분포, 상위 관계 순위
- **설정**: 내 정보, 테마 변경, 로그아웃

---

## 제안 1: 보낸/받은 금액 균형 시각화 (Balance Visualization)

**설명**: 관계별로 보낸 금액과 받은 금액의 균형을 한눈에 파악할 수 있는 시각적 대시보드 위젯. 양방향 수평 바 차트 또는 도넛 차트로 시각화하고, 대시보드에도 "균형이 맞지 않는 관계 Top 5" 위젯을 추가.

**가치**: 한국 경조사 문화에서 "받은 만큼 돌려주는 것"이 중요한 예절. 어떤 관계에서 아직 보답하지 못했는지 직관적으로 파악 가능.

**복잡도**: Medium | **변경 범위**: Frontend + Backend

---

## 제안 2: 경조사 금액 추천 엔진 (Amount Recommendation)

**설명**: 새로운 기록 등록 시, 해당 관계 유형/경조사 유형/과거 이력을 기반으로 적정 금액을 추천. "과거에 받은 금액", "같은 유형 평균 금액", "관계 친밀도에 따른 범위"를 참고하여 가이드라인 제시.

**가치**: 경조사 금액 결정은 한국인에게 가장 고민되는 부분. 과거 데이터 기반 추천으로 의사결정 부담 감소.

**복잡도**: High | **변경 범위**: Frontend + Backend + Database

---

## 제안 3: 실제 캘린더 그리드 뷰 (Calendar Grid View)

**설명**: 현재 일정 관리의 "캘린더" 뷰를 실제 달력 그리드(월간 캘린더)로 개선. 각 날짜 셀에 해당일 일정을 점/뱃지로 표시하고, 클릭 시 상세 확인. 생일/기념일/경조사를 색상으로 구분.

**가치**: 일정의 분포와 밀집도를 한눈에 파악 가능. 특정 주/월에 경조사가 몰려있는지 직관적 확인.

**복잡도**: Medium | **변경 범위**: Frontend only

---

## 제안 4: 일괄 기록 등록 / Excel 가져오기 (Bulk Import)

**설명**: 결혼식/돌잔치 후 여러 건의 축의금/조의금 기록을 한 번에 등록하는 일괄 입력 폼과 Excel/CSV 파일 가져오기.

**가치**: 결혼식 후 방명록의 수십~수백 건 입력 부담 감소. 초기 데이터 입력 허들 대폭 낮춤.

**복잡도**: High | **변경 범위**: Frontend + Backend

---

## 제안 5: 기록 관리 삭제 기능 완성 및 일괄 작업 (Batch Operations)

**설명**: 현재 기록 관리 삭제 버튼에 `// TODO: delete action` 주석이 있어 미구현. 체크박스 선택 후 일괄 삭제/태그 부여/내보내기 작업 완성.

**가치**: 기본 CRUD 완성도 향상. 체크박스 UI가 있으나 동작하지 않으면 UX 불만족 발생.

**복잡도**: Low | **변경 범위**: Frontend + Backend

---

## 제안 6: 데이터 내보내기 / 보고서 생성 (Export & Report)

**설명**: 기록 데이터를 Excel/PDF로 내보내기. "경조사별 축의금 명단", "연간 경조사 지출 요약", "관계별 거래 내역" 보고서 생성.

**가치**: 세금 신고 증빙, 가계부 연동, 기록 백업. 종이 방명록 대비 디지털 관리의 핵심 이점.

**복잡도**: Medium | **변경 범위**: Frontend + Backend

---

## 제안 7: 통계 페이지 차트 고도화 (Chart Enhancement)

**설명**: 현재 단순 수평 바와 텍스트 나열인 통계를 Recharts(이미 설치됨) 활용 차트로 개선. 월별 추이 꺾은선/막대, 유형 분포 파이/도넛, 연도별 비교 그룹 막대 차트.

**가치**: 데이터 시각화 품질이 서비스 전체의 인지 가치를 좌우. 전문적 차트로 신뢰감 향상.

**복잡도**: Medium | **변경 범위**: Frontend only

---

## 제안 8: 기념일 / D-day 알림 설정 (Anniversary Alert Settings)

**설명**: 현재 "준비 중"인 알림 설정 구현. 관계별 기념일/경조사에 대해 "N일 전 알림" 설정. 웹 푸시/이메일/인앱 알림 선택.

**가치**: 경조사를 놓치지 않도록 사전 알림. 서비스 핵심 가치 강화.

**복잡도**: High | **변경 범위**: Frontend + Backend + Database

---

## 제안 9: 관계 프로필 사진 및 아바타 (Profile Avatar)

**설명**: 관계 카드에 프로필 사진 또는 이니셜 기반 아바타 추가. 관계 유형별 기본 아바타 색상 자동 지정, 사진 업로드 가능.

**가치**: 목록이 많아질수록 텍스트만으로는 구별 어려움. 아바타로 직관적 식별.

**복잡도**: Medium | **변경 범위**: Frontend + Backend

---

## 제안 10: 전역 검색 (Global Search)

**설명**: 헤더에 전역 검색 추가. 관계/기록/경조사/일정 전체에서 키워드 검색. 통합 검색 결과 페이지 제공.

**가치**: 데이터 증가 시 필수. 다단계 필터 과정 없이 한 곳에서 바로 검색.

**복잡도**: Medium | **변경 범위**: Frontend + Backend

---

## 제안 11: 경조사 중복 등록 방지 및 연결 (Deduplication & Linking)

**설명**: 동일 경조사 이벤트에 대해 여러 기록을 하나로 그룹화. 등록 시 유사 경조사 자동 제안, 기록-경조사 연결.

**가치**: 하나의 결혼식에서 여러 명에게 축의금을 보내는 경우, 총 지출 파악.

**복잡도**: Medium | **변경 범위**: Frontend + Backend + Database

---

## 제안 12: 다크 모드 지원 (Dark Mode)

**설명**: 라이트/다크 모드 전환 및 시스템 설정 따르기 옵션 추가.

**가치**: 현대 웹 앱 기본 기능. 야간 사용 시 눈 피로 감소.

**복잡도**: Medium | **변경 범위**: Frontend only

---

## 제안 13: 연간/월간 경조사 요약 리포트 (Periodic Summary)

**설명**: 대시보드에 "이번 달/올해 경조사 요약" 카드 추가. 지난달 대비 변화량, 올해 총 지출/수입, 가장 많이 교류한 관계 등 인사이트 제공.

**가치**: 경조사 패턴 인식 및 예산 계획. 사용자 참여도 향상.

**복잡도**: Medium | **변경 범위**: Frontend + Backend
