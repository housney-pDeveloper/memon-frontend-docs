# 품질관리팀 (QA Team)

## 팀 미션

개발 완료된 코드의 품질, 아키텍처 준수, 보안을 종합적으로 검증하여 배포 가능 여부를 판단합니다.

## 담당 워크플로우 단계

**리뷰 → 보안검토**

## 팀 구성

### 필수 에이전트

| 에이전트 | 파일 | 역할 |
|----------|------|------|
| **코드 리뷰 에이전트** | `agent/05_code-reviewer.md` | FSD 규칙, 코딩 컨벤션, 코드 품질, 타입 안전성 검증 |
| **보안 검토 에이전트** | `agent/06_security-auditor.md` | XSS, 인증/인가, 민감 데이터, API 보안, 의존성 취약점 감사 |

### 보조 에이전트

| 에이전트 | 파일 | 역할 |
|----------|------|------|
| **접근성 검토 보조** | `agent/sub_accessibility-checker.md` | WCAG 2.1 AA 기준 접근성 검증, ARIA, 키보드 접근성 |
| **성능 분석 보조** | `agent/sub_performance-analyst.md` | 렌더링 성능, 번들 사이즈, 네트워크 요청, Core Web Vitals |

## 입력 ← 개발팀 산출물

- 변경된 코드 (diff / 파일 목록)
- 대상 프로젝트 (webadmin / appuser / hs-common)
- 자체 검증 결과 (타입 체크, 린트, 테스트)
- 구현 메모 (설계 변경, 제한 사항)

## 검토 우선순위

### 필수 검토 (게이트 통과 조건)

| 검토 항목 | 통과 기준 |
|-----------|-----------|
| FSD 의존성 | 위반 0건 |
| 코딩 컨벤션 | Critical 0건 |
| 보안 취약점 | P0/P1 0건 |
| 타입 안전성 | `any` 사용 0건 |
| API 규칙 | entities(GET만), features(변경) 준수 |

### 권장 검토 (보조 에이전트)

| 검토 항목 | 권장 기준 |
|-----------|-----------|
| 접근성 (WCAG AA) | Critical 0건 |
| Core Web Vitals | LCP < 2.5s, CLS < 0.1 |
| 번들 사이즈 | 이전 대비 20% 이상 증가 없음 |
| 메모리 누수 | cleanup 미비 0건 |

## 플랫폼별 검토 중점

### webadmin

- Full FSD 레이어 규칙 엄격 검증
- ESLint FSD 규칙과의 일치성
- DataGrid, AutoComplete 등 복잡 컴포넌트 검증
- WebSocket 보안 (알림)
- Radix UI primitives 사용 위치 검증

### appuser

- core/ 레이어 사용 여부 (직접 localStorage 접근 금지)
- Radix UI 미사용 확인
- v2(React Native) 전환 호환성
- 모바일 터치 타겟 (44x44px)
- 모바일 네트워크 대응 (로딩 상태)

### hs-common

- 외부 의존성 없음 확인
- 플랫폼 중립성 (브라우저 전용 API 미사용)
- 하위 호환성 유지
- 양쪽 프로젝트 빌드 영향

## 출력 → 배포팀 또는 개발팀

### 통과 시 → 배포팀 전달물

```markdown
## 품질관리 통과 보고서

### 코드 리뷰 결과: 통과
- Critical: 0건
- Major: 0건
- Minor: [N건 (수정 권장)]

### 보안 검토 결과: 통과
- P0: 0건
- P1: 0건
- P2: [N건 (1주 내 수정)]

### 접근성 검토 결과: [통과/참고]
### 성능 분석 결과: [양호/참고]

### 배포 승인: 가능
```

### 반려 시 → 개발팀 반환물

```markdown
## 품질관리 반려 보고서

### 반려 사유
[Critical/P0/P1 이슈 목록]

### 수정 필요 항목
[이슈별 수정 가이드]

### 재리뷰 범위
[수정 후 재검토 대상]
```

## 협업 규칙

1. **코드 리뷰가 1차 게이트** — 코드 리뷰 통과 후 보안 검토 진행
2. **보안 검토와 보조 검토는 병렬** 수행 가능
3. **Critical/P0/P1 이슈 발견 시 즉시 반려** — 수정 후 재리뷰
4. **Minor/Suggestion은 블로킹하지 않음** — 다음 스프린트에서 개선
5. **hs-common 변경은 특별 주의** — 양쪽 빌드 영향 반드시 확인
6. **반려 시 구체적 수정 가이드 제공** — 개발팀이 바로 수정 가능한 수준
