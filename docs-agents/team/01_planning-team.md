# 기획팀 (Planning Team)

## 팀 미션

요구사항을 수집/분석하고, 웹(webadmin)과 앱(appuser) 양 플랫폼에 대한 구체적인 기능 명세를 수립합니다.

## 담당 워크플로우 단계

**요구사항 분석 → 기획**

## 팀 구성

### 필수 에이전트

| 에이전트 | 파일 | 역할 |
|----------|------|------|
| **요구사항 분석 에이전트** | `agent/01_requirements-analyst.md` | 요구사항 수집, 플랫폼 영향도 분석, 기술 실현 가능성 검토, 우선순위 분류 |
| **기획 에이전트** | `agent/02_planner.md` | 기능 명세서 작성, 사용자 스토리 정의, 화면 흐름 설계, 상태 관리 전략 수립 |

### 보조 에이전트

| 에이전트 | 파일 | 역할 |
|----------|------|------|
| **UX 리서처** | `agent/sub_ux-researcher.md` | 사용자 여정 분석, UX 패턴 검증, 사용성 휴리스틱 평가, 접근성 초기 요구사항 |

## 팀 워크플로우

```
[요구사항 입력]
      │
      ▼
┌─────────────────────────────┐
│  요구사항 분석 에이전트       │ ← 1단계: 수집/분석
│  - 기능/비기능 요구사항 분리  │
│  - 플랫폼 영향도 평가        │
│  - 리스크 식별               │
└──────────┬──────────────────┘
           │
           ▼
┌─────────────────────────────┐
│  UX 리서처 (보조)            │ ← 1.5단계: UX 검증
│  - 사용자 여정 분석           │
│  - 사용성 휴리스틱 평가       │
│  - 접근성 초기 요구사항       │
└──────────┬──────────────────┘
           │
           ▼
┌─────────────────────────────┐
│  기획 에이전트               │ ← 2단계: 기능 명세
│  - 기능 명세서 작성           │
│  - 사용자 스토리 정의         │
│  - 화면 흐름 설계            │
│  - 상태 관리 전략            │
└──────────┬──────────────────┘
           │
           ▼
[설계팀으로 전달]
```

## 입력

- 사용자/기획자의 요구사항 (텍스트, 이슈, 화면 설계서)
- 현재 프로젝트 상태 (코드베이스, 기능 목록)
- 디자인 시안 (있는 경우)

## 출력 → 설계팀 전달물

```markdown
## 기획팀 산출물

### 1. 요구사항 분석 결과
- 기능/비기능 요구사항 목록
- 플랫폼 영향도 (webadmin / appuser / hs-common)
- 리스크 및 완화 방안
- 우선순위 (MoSCoW)

### 2. 기능 명세서
- 기능별 상세 명세 (정상/예외 시나리오)
- API 엔드포인트 초안

### 3. 사용자 스토리
- 스토리 및 인수 조건
- 플랫폼별 UX 차이

### 4. 화면 흐름도
- 페이지 구조 및 이동 경로
- 인증/권한 가드 적용 범위

### 5. 상태 관리 계획
- 서버 상태 (TanStack Query)
- 클라이언트 상태 (Zustand)
- 폼 상태 (React Hook Form + Zod)

### 6. UX 리서치 결과 (보조)
- 사용자 여정 분석
- 사용성 개선 제안
- 접근성 요구사항
```

## 협업 규칙

1. 요구사항 분석 에이전트가 **먼저** 분석을 완료한 후, 기획 에이전트가 작업을 시작한다
2. UX 리서처는 요구사항 분석과 기획 **사이**에 투입되어 UX 관점의 검증을 수행한다
3. 웹/앱 동시 대상 기능은 반드시 양 플랫폼 UX 차이를 명시한다
4. appuser의 v2(React Native) 전환 계획과의 호환성을 항상 고려한다
5. 산출물은 설계팀이 바로 작업을 시작할 수 있는 수준으로 구체화한다
