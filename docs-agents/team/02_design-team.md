## 설계 원칙

### webadmin (Full FSD)

```
app → widgets → features → entities → shared
```

- entities: read-only (GET), features: write (POST/PUT/PATCH/DELETE)
- 교차 참조 금지 (entities 간, features 간)
- barrel import 금지
- primitives는 shared/ui에서만

### appuser (Light FSD)

```
app → features → shared (+ core/)
```

- core/: 플랫폼 추상화 필수 (v2 전환 대비)
- Radix UI 미사용 (순수 Tailwind)
- 직접 localStorage/window 접근 금지

### 공통 원칙

- `buildApiUrl()` 사용 (URL 패턴 통일)
- `ServerResponse<T>` 응답 타입 사용
- 에러는 서버 메시지 그대로 표시
- Arrow Function, Props 분리, displayName 필수

## 협업 규칙

1. 설계 에이전트가 **전체 아키텍처를 먼저** 확정한 후, API 설계 보조가 상세 작업에 진입한다
2. API 설계 보조와 컴포넌트 상세 설계는 **병렬 수행** 가능하다
3. 설계 결과는 ESLint FSD 규칙과의 호환성을 반드시 검증한다
4. hs-common 변경이 있으면 양쪽 프로젝트 영향도를 반드시 명시한다
5. 개발팀이 구현 순서를 바로 결정할 수 있도록 의존성 정보를 포함한다
